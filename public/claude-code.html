Introduction

What is Claude Code?

An AI agent that lives in your terminal and works like a capable engineer — not a line-by-line autocomplete.

  • Reads and writes files across your entire project
  • Runs shell commands — tests, builds, git, linters
  • Works autonomously on multi-step tasks end-to-end
  • Loads project context from CLAUDE.md every session

Not a plugin

Claude Code is a complete autonomous agent — it plans, acts, and verifies. It can touch 50 files in one instruction.

## WHAT — Stack & Structure
Monorepo: apps/api, apps/web, apps/worker
Shared packages: packages/db (Prisma),
packages/auth (JWT), packages/ui (React).
API: Express + TypeScript. Web: Next.js.
Worker: long-running BullMQ consumers.
DB: Postgres. Cache: Redis. Queue: RabbitMQ.

## WHY — Purpose & Domain
B2B payments platform. Merchants create
accounts, connect Stripe, and process
refunds/payouts via the API. The worker
handles async webhook delivery and retry
logic. The web app is the merchant dashboard.
Payment states: pending → processing
→ settled | failed | refunded.
"amount" is always cents (integer).

## HOW — Working on this Project
Use bun, not node/npm.
bun test # run all tests
bun run typecheck # tsc --noEmit
bun run lint # eslint + prettier
Always typecheck before marking done.
Never call Stripe API in tests — use
fixtures in tests/fixtures/stripe/.
Never edit applied migration files.

Commands

/rewind

Claude checkpoints as it works. If it goes sideways, /rewind rolls back to any prior state — you don't lose the whole session, just the part that went wrong.

Commands

/btw

Ask quick questions without adding them to your conversation history. Claude answers from context, the exchange disappears, and your main thread stays clean — even while Claude is mid-response.

Commands

/find-bugs

Scan the current branch for bugs, security vulnerabilities, and code quality issues. Claude reviews only the changes you've made, not the entire codebase.

Commands

Permission modes

Tune exactly how much Claude asks before acting. Press Shift+Tab to cycle modes mid-session.

.claude/settings.json
{
 "permissions": {
  "allow": [
   "Bash(npm run test)",
   "Bash(npm run lint)",
   "Bash(git status)",
   "Bash(git diff *)",
   "Bash(git commit *)"
  ],
  "deny": [
   "Bash(git push *)",
   "Bash(npm install *)",
   "Bash(rm -rf *)",
   "Edit(.env*)"
  ]
 }
}

Modes — Shift+Tab to cycle
default asks before every edit and command
auto-accept edits file edits silent, commands still prompt
plan mode read-only, review plan before anything runs

Advanced usage

Orchestrating agents with CLAUDE.md

.claude/CLAUDE.md

## Agent Roles
# Planner — reads PVR, writes test_plan.md
# Coder — generates .casm from plan
# Debug — fixes dvsim failures
# Critic — gates on style + correctness

## Workflow
1. Invoke `/directed-test-migration-agent`
2. Planner reads PVR → test_plan.md
3. Coder runs per-PVR in parallel
4. dvsim gate → Debug on FAIL
5. Critic approves → directed_tests/
1

Skill invocation kicks off the orchestrator — no manual wiring needed

2

Planner uses dv-reference skill to produce a grounded test plan

3

One Coder agent spawned per PVR — all run simultaneously

4

dvsim output gates the loop — Debug agent retries automatically on FAIL

5

Critic reviews style + correctness before writing to directed_tests/
