Where the speed comes from
To make one user's tokens faster, you split the model across more GPUs. Call that number k — split across k GPUs and each one reads only 1/k of the weights.
time per token = read ÷ k + sync
read ÷ k
Each GPU reads its 1/k slice in parallel. This is the speedup.
sync
After every layer the GPUs stop to combine results. Add GPUs and this grows.
The work never shrinks. You read the whole model either way, and now you pay for the sync on top. Faster tokens cost more energy.
